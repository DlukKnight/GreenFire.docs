    Dashboard runtime context

    Description

    DashboardLayouts have their own runtime context, being simpler than the regular runtime context, since they don't use more complex scenarios, like validations.

    Context properties

    Dashboard context

    dashboardCode

    Identifies the Dashboard owning the layout.

    serviceUrl

    The base Url for the service that will be passed to child nodes, so they can correctly retrieve data if needed.

    Layout context

    dashboardLayout

    The DashboardLayout definition necessary to render the layout.

    dashboardState

    The Dashboard filters form data.

    Dashboard display state

    disabled

    When true, the layout is disabled and cannot be interacted. It also changes the visual style of all child nodes to reflect it.

    isLoading

    Represents the readiness state of the owning Dashboard. If it is true, then the Dashboard should be not ready still to operate, therefore, all child nodes should be disabled to prevent inconsistent behavior.
