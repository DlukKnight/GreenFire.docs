# **Overview**

## **Description**

GreenFire provides several components to be used, from basic components like [**textboxes**](./controls/textboxcontrol.md), to more complex ones, like [**Json controls**](./controls/rootlayoutjsoncontrol.md), and culminating in full processes, like [**Crud Processes**](./processes/crudprocess.md) or [**Dashboards**](./processes/dashboard.md).

Most of the time, layouts will be defined inside a [**RootLayout**](./containers/layout/rootlayoutcontainer.md) node, which will have all the information to render the full layout. Similarly, Dashboards have their own layout container, called [**DashboardLayout**](./containers/dashboard/dashboardlayoutcontainer.md)

These components are meant to work with metadata, with our recommended way being the use of a .NET API using the GreenFire.Net package.
